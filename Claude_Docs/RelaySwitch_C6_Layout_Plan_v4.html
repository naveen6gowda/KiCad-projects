📐 The Width Problem & Solutions

🔴 The Core Issue: HLK + Relays Width

With your exact dimensions, placing HLK and relays in the same row:

  • HLK(30w) + 1mm gap + K1(21w landscape) = 52mm → 2mm over the 50mm board!
  • HLK(30w) + K1(11.5w portrait) + K2(11.5w portrait) = 53mm → 3mm over

There are 3 solutions, ranked by impact:

✅ Solution A (BEST): Rotate HLK-2M05 to Portrait Orientation

If the HLK module can be oriented with the 16mm side as width and 30mm as height:

  • HLK width = 16mm instead of 30mm
  • 16mm(HLK) + 1mm + 21mm(K1 landscape) = 38mm → fits easily with 12mm spare!
  • Or: 16mm(HLK) + 1 + 11.5(K1) + 1 + 11.5(K2) = 41mm → also fits!
  • HLK height becomes 30mm → occupies Y=0 to Y=30mm, almost the entire HV zone
  • Connectors J1, J3, J4 sit to the RIGHT of HLK, stacked vertically

Check in KiCad: Select U1 (HLK-2M05), press R to rotate 90°. Verify that the pin numbering still matches your schematic. The AC input pins should face the board edge.

⚠️ Solution B: Zero-Gap + Route Slot

Keep HLK at 30mm width, place relays at X=30mm (0mm gap) with a routed isolation slot between HLK and relay pads. The slot provides creepage distance without wasting board width.

30(HLK) + 0 + 11.5(K1) + 0 + 11.5(K2) = 53mm — still 3mm over.

This only works if K1 and K2 overlap the HLK slightly or you use landscape relays: 30 + 0 + 21(K landscape) = 51mm — 1mm over. Marginal — might work with courtyard overlap.

⚠️ Solution C: Increase Board to 55×45mm

Adding 5mm width gives: 30(HLK) + 2 + 21(K1 landscape) + 2 = 55mm → exact fit. Or 30 + 1 + 11.5 + 1 + 11.5 = 55mm. This is the no-compromise solution but changes your board size target.

🏆 Solution A: Detailed Layout — HLK Rotated to Portrait

⚠️ ESP32 Height vs LV Zone

Your ESP32 module with THT headers is 17.8mm tall, but the LV zone (below isolation at Y=31mm) is only 14mm tall (45 - 31 = 14mm). The ESP32 doesn't fit in the LV zone alone!

Options:

  • Option 1 (RECOMMENDED): Move isolation line to Y=27mm (from top). This gives LV zone = 18mm height (enough for 17.8mm ESP32). HV zone becomes 27mm — tight but works if HLK is portrait: connectors(12.6) + relays(11.5 landscape) = 25.1mm, leaving 1.9mm for traces.
  • Option 2: Use castellated SMD pads instead of THT headers for ESP32. The bare module is only 21×17.8×2.4mm. Soldered flat on B.Cu, it's only 2.4mm thick. This frees the top side completely and the module fits on B.Cu under the HV zone.
  • Option 3: Increase board to 50×50mm. Extra 5mm height solves all fitting issues. LV zone becomes 19mm.

🏆 My Strongest Recommendation: Castellated ESP32 on B.Cu

The single change that solves ALL your fitting problems is switching from THT pin headers to castellated/SMD pads for the ESP32-C6. The XIAO module was designed for this — it has castellated edges specifically for direct soldering to a carrier PCB.

  • ESP32 moves to B.Cu (bottom side), under the HV zone — no height conflict
  • LV zone on top becomes FREE for J5, J6, J2 only — plenty of room
  • All SMD passives surround the ESP32 on B.Cu — shortest traces
  • The isolation barrier still works — ESP32 on B.Cu stays below the isolation line in LV zone
  • Board height crisis eliminated — no 17.8mm component fighting for 14mm space

In KiCad: change the ESP32-C6 footprint from the THT header version to the castellated/SMD pad version. The XIAO KiCad library has this footprint (surface mount, bottom-side pads).
