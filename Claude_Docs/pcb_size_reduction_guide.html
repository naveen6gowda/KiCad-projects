📐 The Aqara T2's outer enclosure is 45.5×48mm — the actual PCB inside is approximately 38×42mm. The enclosure adds ~7mm all around. Your target of 45×50mm for the PCB is very achievable with the right component choices below.

2. How Aqara Achieves That Size — Their Secrets

🔌 No Traditional PSU Module

Aqara does NOT use an HLK-PM01 module. They use a capacitor-dropper or tiny SMPS IC (like BP2525, BP2528 or similar single-chip mains-to-3.3V converter) directly on the PCB. These chips are 1–2mm SOP packages vs your HLK-PM01 which is 34×20×15mm by itself — the single biggest space hog on your board.

📡 Integrated SoC — No Separate Module

Aqara uses the EFR32MG24 — a bare SoC chip (5×5mm QFN) with Zigbee built in. They don't use a development module like XIAO ESP32-C6 (which is 17.5×21mm DIP with PCB, USB port, LEDs, buttons all adding bulk you don't need).

⚡ SMD Relays (or NO relays at all)

Your HF3FF relay is 19×15mm per unit × 2 = 38×15mm just for relays. Aqara uses either much smaller relays (like Omron G5Q = 12.5×7.5mm) or in some models uses triacs/SSRs instead of mechanical relays — no moving parts, tiny SMD package.

🔧 Zero Through-Hole Parts

You have THT relays, THT transistors (TO-92), THT diodes (DO-41), THT optocouplers (DIP-4). Each THT component needs 2.54mm pin spacing, large drill holes, and keep-out zones. Aqara uses 100% SMD — every component is surface mount, allowing much denser packing.

5. Relay Swap — HF3FF → G5Q

❌ Current: HF3FF (19×15mm each)

  • 19.0 × 15.2mm footprint
  • THT 5-pin (large drill holes)
  • Height: 15.5mm tall
  • Two of these = 38×15mm strip
  • Coil: 5VDC, 360mW

✓ Recommended: Omron G5Q-1 (12.5×7.5mm)

  • 12.5 × 7.5mm footprint
  • THT, same 10A/250VAC rating
  • Height: 10mm tall (lower profile)
  • Two of these = 25×7.5mm
  • Coil: 5VDC, 200mW (less heat!)
  • KiCad: Relay_THT:Relay_SPDT_Omron_G5Q
💡 The G5Q is a direct electrical equivalent to HF3FF — same 10A 250VAC rating, same SPDT contact form, same 5V coil. It's just a physically smaller "sugar cube" style relay. No schematic changes needed, just swap the footprint in KiCad.

6. MCU Module Swap

❌ Current: XIAO ESP32-C6 DIP (21×17.5mm)

The XIAO is a great dev tool but includes USB-C port, LEDs, reset button, 5V regulator — things you don't need in a final product. All of these add bulk.

✓ Recommended: ESP32-C6-MINI-1 (13.2×16.6mm)

Bare ESP32-C6 module. Same chip, same Zigbee/WiFi, smaller PCB footprint, castellation pads for SMD soldering. Official Espressif module, CE/FCC certified for commercial use.

7. Step-by-Step Action Plan to Hit 45×50mm

  • 01
    Swap all THT passives to SMD — Diodes 1N4007 → SMA package. Transistors S8050 → MMBT8050 SOT-23. Optocouplers PC817 DIP → PC817S SMD. This alone reclaims huge area and removes the need for large drill keepouts. saves ~300mm²
  • 02
    Replace HF3FF relays with Omron G5Q-1-DC5 — Same rating, smaller body. Place both relays side by side along one edge of the board. saves ~380mm²
  • 03
    Switch from XIAO to ESP32-C6-MINI-1 — Reuse all your schematic logic, GPIO assignments stay the same. Update the symbol and footprint in KiCad. saves ~150mm²
  • 04
    Replace HLK-PM01 with HLK-PM01L (bare board) for prototyping. Plan for a BP2528-based discrete SMPS for production. saves ~500mm²
  • 05
    Use 3.5mm pitch screw terminals instead of 5.08mm — Many Indian panel connector suppliers carry 3.5mm pitch terminal blocks rated for 10A. Saves ~30% connector length.
  • 06
    Use SMD fuse 1206 (Littelfuse 0452002.MR) instead of any through-hole fuse holder. Already discussed — tiny footprint.
  • 07
    Place components on both sides of the PCB — Put the ESP32-C6 module and LV passives on top, put the relays and HV components on the bottom. KiCad supports this with the Fab_Back and Cu_Back layers. This is standard for commercial smart switches.
  • 08
    Eliminate the debug UART header J1 — Add programming pads (test points) instead of a full connector. For production boards, a 6-pad UART footprint with no connector saves ~15×10mm.
