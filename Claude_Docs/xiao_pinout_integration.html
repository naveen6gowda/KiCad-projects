KiCad Integration Guide — XIAO-ESP32-C6-DIP
⚠ USE DIP VARIANTSelect XIAO-ESP32-C6-DIP from the Seeed library. The DIP variant matches the castellated pad footprint of the XIAO module.

XIAO ESP32-C6 DIP (SELECT THIS)

  • Castellated/through-hole pads
  • Pads along both long edges
  • Module sits on top of PCB
  • Easy hand-soldering
  • Matches SuperMini form factor
  • Has antenna keepout zone

XIAO ESP32-C6 SMD (avoid)

  • Bottom-only SMD pads
  • Requires reflow oven
  • Different pin arrangement
  • For direct IC mounting only
  • Not suitable for module
  • Hidden solder joints
Step-by-Step KiCad Symbol & Footprint Setup
Add Seeed Library to KiCad
In KiCad → Preferences → Manage Symbol Libraries → Add:
Seeed_Studio_XIAO_Series.kicad_sym Same for footprint: Manage Footprint Libraries → Add:
Seeed_Studio_XIAO_Series.pretty
Place Symbol in Schematic
Press A → Search "XIAO-ESP32-C6-DIP" → Place U1.
The symbol shows all 22 exposed pads in two rows.
Assign Footprint
Double-click U1 → Footprint field → Click browse → Select:
Seeed_Studio_XIAO_Series:XIAO-ESP32-C6-DIP Confirm pad count = 22 (11 each side) matches.
Add Power Symbols
Connect pin 3V3 → power symbol +3.3V
Connect pin 5V → power symbol +5V
Connect all GND pins → power symbol GND
Add 10kΩ pull-up from +3.3V to EN pin.
Add Net Labels for Relay Control
GPIO0 → add net label: RELAY_CH1
GPIO1 → add net label: RELAY_CH2
These nets connect to PC817 optocoupler anodes via 1kΩ resistors.
Add Net Labels for Energy Monitor
GPIO20 (D9/MISO) → net label: HLW_CF
GPIO21 (D3) → net label: HLW_CF1
GPIO22 (D4/SDA) → net label: HLW_SEL
Add 10kΩ pull-up resistors on CF and CF1 to +3.3V.
Mark Unused Pins
GPIO12/13 (USB D-/D+): Add "Do Not Connect" X markers — these are reserved for USB.
GPIO14 (RF Switch): Add X marker — used internally by module.
GPIO15 (User LED): Can leave free or connect to onboard LED.
PCB Layout — Antenna Keepout
After import to PCB editor, add a courtyard keepout zone:
Board Setup → Constraints: set 5mm keepout at antenna end (USB-C side) No copper fills, vias or traces within this zone on ANY layer.
Verify Footprint Dimensions
XIAO ESP32-C6: 22.52 × 17.78mm, pad pitch 2.54mm
If your SuperMini differs slightly, adjust the footprint pad positions:
Footprint Editor → move pads to match SuperMini datasheet

GPIO Assignment Summary — Smart Relay Design
RELAY CONTROL
GPIO0 → D0/A0 → RELAY_CH1
  → 1kΩ R1 → PC817 U4 Anode
  → Logic: LOW=ON, HIGH=OFF

GPIO1 → D1/A1 → RELAY_CH2
  → 1kΩ R3 → PC817 U5 Anode
  → Logic: LOW=ON, HIGH=OFF
ENERGY MONITORING
GPIO20 → D9/MISO → HLW_CF
  → Power frequency pulse output
  → 10kΩ pull-up to 3.3V

GPIO21 → D3 → HLW_CF1
  → V or I pulse (SEL selects)
  → 10kΩ pull-up to 3.3V

GPIO22 → D4/SDA → HLW_SEL
  → 0=Current, 1=Voltage
UART / PROGRAMMING
GPIO16 → D6 → TX
  → UART0 TX (programming)
  → 3-pin header J5

GPIO17 → D7 → RX
  → UART0 RX (programming)
  → 3-pin header J5
I2C / EXPANSION
GPIO4 → MTMS/A4 → SDA
  → I2C SDA (expansion opt.)

GPIO5 → MTDI/A5 → SCL
  → I2C SCL (expansion opt.)

GPIO2/3 → D2/D11
  → Expansion relay CH3/CH4
DO NOT USE — RESERVED
GPIO12 → USB D-
GPIO13 → USB D+
GPIO14 → RF Switch
GPIO15 → User LED
GPIO8 → WS2812 (onboard)
GPIO9 → BOOT button
FREE GPIO (EXPANSION)
GPIO6 → MTCK/A6 → Free
GPIO7 → MTDO → Free
GPIO18 → D10/MOSI → Free
GPIO19 → D8/SCK → Free
GPIO23 → D5/SCL → Free

5 GPIO pins available for LEDs,
buttons, sensors, extra relays

# ESPHome config for XIAO ESP32-C6 Smart Relay
esp32:
  board: esp32-c6-devkitc-1
  framework:
    type: esp-idf

# Relay Channel 1 — GPIO0 (XIAO D0)
switch:
  - platform: gpio
    pin: GPIO0
    name: "Relay Channel 1"
    inverted: true # active-LOW via PC817

  - platform: gpio
    pin: GPIO1
    name: "Relay Channel 2"
    inverted: true

# HLW8012 Energy Monitor
sensor:
  - platform: hlw8012
    sel_pin:
      number: GPIO22 # XIAO D4
      inverted: true
    cf_pin: GPIO20 # XIAO D9
    cf1_pin: GPIO21 # XIAO D3
    current_resistor: 0.001
    voltage_divider: 940
    current:
      name: "Current"
    voltage:
      name: "Voltage"
    power:
      name: "Power"
    update_interval: 10s
