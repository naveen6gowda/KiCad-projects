3. Your Three Real Options

🏆 OPTION A: Castellated ESP32 on B.Cu

Change ESP32 footprint from THT headers to castellated SMD pads.

ESP32 goes on BOTTOM side, soldered flat. Only 2.4mm thick. This removes 17.8mm from the height budget entirely.

New height: 12.6(conn) + 1 + 16(HLK) + 3(iso) + 0(ESP32 on B.Cu) = 32.6mm out of 45mm → 12.4mm spare ✓

Board stays 45×50mm. No other changes needed.

This is how Aqara/Sonoff do it — module soldered flat to carrier PCB.

OPTION B: Bigger Board

Increase board to 50×55mm (add 10mm height).

Extra 10mm gives LV zone = 22mm height for THT ESP32 (17.8mm) + gaps.

Keeps THT ESP32 headers for easy development/debugging.

Trade-off: may not fit target Indian switchboard module size.

OPTION C: Smaller PSU

Replace HLK-2M05 (30×16mm) with HLK-PM01 (24×18×15mm) or a bare SMPS IC design.

Saves 6mm width but height problem remains: 12.6+18+3+17.8 = 51.4mm → still over.

Doesn't solve the height problem alone. Combine with Option B for best result.

OPTION D: Overlap ESP32 into HV zone

Remove isolation requirement between ESP32 and HV components.

ESP32 body overlaps relay zone. HV/LV isolation maintained only through PCB traces (not physical distance).

Risk: reduces safety margin. Optocouplers still provide galvanic isolation for relay control. Voltage divider resistors provide isolation for HLW8012. May work if you route carefully, but not recommended for production.

📋 How to Switch to Castellated in KiCad

  1. Open your schematic → find U2 (XIAO ESP32-C6)
  2. Double-click → Edit Footprint → change to the SMD/castellated variant from the XIAO KiCad library (SeeedStudio provides this)
  3. If you don't have the castellated footprint: download from github.com/Seeed-Studio/OPL_Kicad_Library
  4. The castellated footprint has half-moon pads on the board edge instead of through-holes
  5. In PCB editor, place U2 on B.Cu (press F to flip before placing)
  6. The module will solder directly onto the bottom side — no headers needed
  7. This is how commercial products (Aqara, Sonoff) mount their MCU modules
